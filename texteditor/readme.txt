viva Text Editor에 대한 설명서 입니다.
기본 파일 설명
 ncurses를 사용하여 만들었으며, windows는 ncurses 라이브러리를 지원하지 않기 때문에 windows에서 ncurses를 사용하기 위해 lib파일에 pdcurses.a, pdcurses.lib 파일을 넣고 curses.h를 통해 windows환경에서도 ncurses를 사용할 수 있도록 하였습니다. 또한, 리눅스와 맥에서는 ncurses 라이브러리를 지원하며, 사용 시에는 미리 ncurses라이브러리를 다운로드 후 사용하여야 합니다. makefile을 이용해 make를 실행하면 각 운영체제에 맞는 실행 파일이 생성 됩니다. 

파일 실행
 새로운 파일을 생성할 경우에는 ./viva를 사용하여 새로운 파일을 생성이 가능하며, 간혹 윈도우에서는 viva형태로 사용해야하는 경우도 있습니다.
기존에 있는 파일을 열 경우에는 ./viva viva.c 처럼 ./viva 기본파일 형태를 사용하면 기존 파일을 열 수 있습니다. 위처럼 간혹 윈도우에서는 viva를 사용해야하는 경우에는 viva viva.c 즉 viva 기본파일 형태를 사용해 열 수 있습니다.
리눅스와 맥에서는 사용자 권한으로 인해 make가 실행되지 않거나 파일 저장에 문제가 생길 수 있으므로 sudo를 붙여서 사용하면 실행이 가능합니다. (예로 make가 안되면 sudo make, viva가 안열리고 파일저장이 안되면 sudo ./viva를 사용해서 하면 파일이 실행됩니다. 기존 파일을 들고 올 때 또한 sudo ./viva 기본파일 형태로 사용하면 됩니다.)

파일이 실행되면 아래와 같은 기능이 작동됩니다.

상태 바
 - 아래에서 2번째 라인에 반전 색상으로 나타납니다.
 - 파일 이름, 라인 수, 파일 타입, 현재 커서가 있는 위치를 표시합니다.
 - 파일 이름은 새로운 파일을 생성 했을 시에 no name으로 나타나며 저장을 통해 파일이름을 작성하면 변경되고 기존 파일을 열 경우에는 기존 파일을 이름을 가지고 오게 됩니다.
 - 라인 수는 엔터키를 눌러 새로운 라인을 생성하면 늘어나고 백스페이스를 통해 새 라인을 지우면 라인 의 수가 줄어듭니다.
- 파일 타입은 새로운 파일을 생성 시에 no ft로 가져오며 저장시 예시로 text.txt처럼 파일 타입을 붙여서 저장하면 .txt처럼 . 파일 타입 형태로 가져오게 되며, 파일 타입을 붙이지 않고 text 처럼 저장하면 no ft를 그대로 출력합니다. 또한, 기존 파일을 가져올 시 파일 타입이 없으면 no ft로 출력되고 파일 타입이 있다면 .파일타입 형태로 출력됩니다.
- 현재 커서가 있는 위치는 현재 커서의 라인 위치 / 라인 수 의 형태를 띄고 있으며 새로운 파일 생성 처음에는 문자가 들어오지 않았기에 1/0으로 표기 됩니다. 

메시지 바
 - 가장 아래 라인에서 나타납니다.
 - 처음과 텍스트 수정 때는 메시지 바가 저장, 탐색, 나가기 방법을 HELP: Ctrl-S = save | Ctrl-Q = quit | Ctrl-F = find 로 나타냅니다.
 - Ctrl-s가 입력되었을 때 만일 새 파일을 저장하는 단계라면 저장, 탐색, 나가기 방법에 대해 적힌 내용이 사라지고 파일의 이름을 입력 받을 수 있도록 됩니다.
 - Ctrl-F가 입력되었을 때 문자를 찾기 위해 저장, 탐색, 나가기 방법에 대해 적힌 내용이 사라지고 단어 검색이 가능합니다. 검색을 했을 경우 검색한 문자를 띄워 둔 상태가 됩니다.
-  다른 단어를 찾을 땐 이전 단어가 사라지고 새로 입력 받을 수 있는 상태가 됩니다.

텍스트 수정
 - 새 파일이 생성시 커서가 x, y의 좌표가 0,0에서 입력을 받도록 합니다.
 - 기존 파일을 불러오면 header가 가리키는 연결리스트 속 배열의 index 0로 가고 커서는 x, y의 좌표가 0,0에서 입력 받도록 됩니다.
 - 문자를 입력하는 키를 누르면 문자가 작성 됩니다.
 - 엔터를 누르면 연결리스트가 새로 생성되며 뉴 라인이 생성 되고 문자를 입력받습니다.
 - 만일 연결리스트 속 배열에 저장된 문장의 문자들이 출력되는 화면 창의 가로 넓이보다 많다면 배열에서는 한 연결리스트에 문자들이 저장되어 있지만 출력 화면에서는 문자의 양과 화면 창의 가로넓이에 맞춰 2줄이상으로 표기되도록 하였습니다. 
 - 백 스페이스를 누르면 문자가 삭제되며 빈 라인이거나 문장의 맨 앞이라면 라인이 삭제 됩니다.

텍스트 수정 과정중에 이동이 필요할 때 입니다.
 
화살표 키
 - 오른쪽 화살표를 누르면 커서가 오른쪽으로 움직이고 문장의 맨 끝이라면 다음 문장의 맨 앞으로 넘어갑니다.
 - 왼쪽 화살표를 누르면 커서가 왼쪽으로 움직이며 문장의 맨 앞으로 오면 이전 문장의 끝으로 이동합니다.
 - 위쪽 화살표를 누르면 커서가 위로 움직이며 만일 화면에서 2줄로 표기되고 2번째 줄에 있다면 1번째줄로 이동하고 화면에서 현재 문장의 첫번째 줄에서 위쪽화살표를 누르면 이전 문장으로 이동됩니다. 이때, 이전 문장의 맨 뒤 내용이 현재 커서의 x좌표보다 작다면 x좌표가 이동됩니다.
 - 아래쪽 화살표를 누르면 커서가 아래로 움직이며 만일 화면에서 2줄로 표기되고 1번째 줄에 있다면 2번째 줄로 이동하고 화면에서 현재 문장의 맨 아랫줄에 있다면 다음 문장으로 이동됩니다 이때, 다음 문장의 내용이 x좌표가 있는 위치보다 작다면 x좌표가 이동하게 됩니다.

홈 키
 - 현재 커서가 있는 위치의 x좌표가 0이되는 곳으로 이동 시킵니다.

엔드 키
  - 현재 커서가 있는 위치에서 문장이 출력하는 화면보다 내용이 적을 경우 맨뒤로 이동되고 내용보다 많은 경우는 x좌표가 출력화면의 가로 맨 끝으로 이동합니다.

페이지 업 키
 - 현재 커서가 있는 위치에서 출력화면의 row길이 만큼 위로 이동해 줍니다. 이때, 이동된 위치는 출력화면의 제일 위에서 부터 출력됩니다.

페이지 다운 키
 - 현재 커서가 있는 위치에서 출력화면의 row길이만큼 아래로 이동해 줍니다. 이때, 이동된 위치는 출력화면의 제일 위에서 부터 출력됩니다.

저장 키
 - Ctrl-S키를 사용하면 파일이 저장 가능합니다.
 - 새 파일인 경우 저장 시 파일을 이름을 메세지 바에서 받게됩니다. 메세지 바에서 받은 이름을 토대로 파일을 생성합니다.
 - 기존 파일일 경우나 새로운 파일을 한 번 이상 저장 했다면 Ctrl-S를 누르면 파일 이름을 받을 필요없이 바로 저장이 가능합니다.

탐색 키
 - Ctrl-F를 누르면 원하는 문자를 입력하여 문장에서 찾을 수 있습니다. 
 - Ctrl-F키를 누르면 메시지 바에서 찾고자하는 문자를 받을 수 있습니다.
 - 원하는 문자를 작성 후 엔터를 누르면 header에서 부터 탐색에 들어가게 됩니다. 이후 일치하는 문자가 있다면 하이라이트로 표시후 찾고자 한 문자의 앞으로 커서가 이동됩니다.
 - 찾고자 한 문자에서 오른쪽 화살표를 누르면 현재 줄에 중복적으로 같은 단어가 있는지 확인 후 있다면 그 위치로 이동 후 찾았던 문자는 하이라이트가 사라지고 다시 찾은 문자에 하이라이트가 생성 됩니다. 현재 줄에 중복적으로 단어가 없다면 다음 줄에서 문자를 찾습니다. 마지막 문장까지 탐색 했을 때 없다면 더이상 오른 쪽으로 이동이 불가능 합니다. 
- 찾고자 한 문자에서 왼쪽을 누르면 제일 처음 찾은 문자가 아니라면 현재 문장의 현재 위치보다 앞에 찾고자한 문자가 있는지 탐색 후 있다면 그 문자를 하이라이트 후 커서를 이동하고 없다면 이전 문장으로 이동하여 탐색 후 있다면 그 문자를 하이라이트 후 커서를 이동합니다. 제일 처음 찾은 문자라면 더이상 이동이 불가능 합니다.

나가기
 - Ctrl-Q를 누르면 프로그램을 종료합니다.
 - 만일 파일이 저장된 뒤 Ctrl-Q를 한 번 눌렀다면 바로 프로그램을 종료할 수 있다. 허나, 한 글자라도 적거나 지운다면 프로그램을 바로 종료 할 수 없다. 이때, 파일을 저장하지 않고 나가기 위해서는 Ctrl-Q를 두번이상 눌러야 한다.
 - 새로운 파일을 생성하고 아무것도 적지 않고 나가려 하려 해도 Ctrl-Q를 2번 이상 눌러야 저장 없이 나갈 수 있다.
 - 기존의 파일을 열었을 경우 문자를 입력하거나 삭제하지 않았다면 Ctrl-Q를 한 번만 누르고 저장 없이 나갈 수 있다. 허나, 한 글자라도 적거나 삭제 했다면 Ctrl-Q를 2번 이상 눌러야 저장 없이 나갈 수 있다.
